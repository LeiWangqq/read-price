# 文件快速定位工具

在大批 PDF / Word 文档（含扫描件）中快速定位某个内容**在哪个文件、第几页、第几章**，并返回原文片段。

适用场景：造价/采购人员在数十甚至上百份招标文件、价格台账、合同中查找某个材料单价出现在哪里。

---

## 功能概览

### 文档支持
| 格式 | 说明 |
|------|------|
| PDF（数字） | Word/WPS 直接生成的 PDF，直接提取文字 |
| PDF（扫描） | 纸质文件扫描件，自动 OCR 识别中文 |
| PDF（混合） | 部分页有文字、部分页为扫描图，自动逐页判断 |
| Word (.docx) | 直接解析，按章节标题分块 |
| Word (.doc) | 旧版 Word 格式，自动转换后解析 |

### 核心功能
- **智能定位**：输入关键词（如"防腐涂料的单价"），返回包含该内容的文件名、页码、章节、原文片段。
- **多关键词 AND 匹配**：输入"防腐涂料 单价"时，只有同时包含两个词的文本块才会命中，减少无关结果。
- **缓存版本管理**：每次索引结果保存为独立的时间戳版本（如 `2026-06-08_15-18-42`），旧版本不丢失，可随时加载任意历史版本。
- **AI 整理**（可选）：接入大模型（OpenAI 兼容接口），将搜索结果自动整理为结构化表格，流式输出。
- **关键词高亮**：搜索结果中关键词自动高亮标记，快速定位关键信息。

### 输入方式
- **本地文件夹**：指定路径，递归扫描其中所有 PDF/Word 文件。
- **上传文件**：通过网页拖入或选择文件上传。
- **单个文件**：输入或浏览选择单个文件路径。

## 使用步骤

### 1. 首次启动

双击 `setup.bat`（需联网，约 2-3 分钟），自动下载依赖到 `libs/` 目录。
之后双击 `start.bat`，自动打开浏览器 `http://localhost:8501`。

### 2. 配置大模型（可选）

在左侧边栏 **① 大模型配置** 中填写：
- **Base URL**：大模型 API 地址（如 `https://api.example.com/v1`）
- **API Key**：密钥
- **Model**：点击"获取模型列表"自动拉取，或手动填写

> 不配置大模型也可以使用搜索功能，只是结果不会经过 AI 整理。

### 3. 加载已有缓存

在 **② 缓存提取** 中选择加载方式：
- **从最新缓存加载**：选择文件夹或单个文件路径，加载最近一次索引结果。
- **从历史缓存加载**：下拉选择任意历史版本，查看或加载。

### 4. 建立索引

在 **③ 文档来源** 中选择文档来源：
- **本地文件夹**：点击"浏览选择文件夹"或手动输入路径，点击"扫描并建立索引"。
- **上传文件**：拖入 PDF/Word 文件，点击"建立索引"。

索引过程会自动完成：
- 文本提取（数字 PDF / Word）
- OCR 识别（扫描页，自动检测）
- 结果缓存（带时间戳版本）

### 5. 查询

在 **④ 输入查询** 中输入关键词，点击"查询"。

结果展示：
1. **总览表格**：文件名 + 命中页码，一目了然。
2. **文件详情**：按文件分组展开，显示页码、章节、原文片段。
3. **AI 整理**（如已配置）：自动将结果整理为表格格式，流式输出。

---

## 搜索逻辑

- **关键词提取**：自动去除"在哪个文件""第几页"等停用词，提取核心关键词。
- **连接词拆分**：输入"防腐涂料和单价"会被拆分为"防腐涂料"和"单价"两个独立关键词。
- **AND 匹配**：多个关键词必须同时出现在同一文本块中才会命中。
- **大小写敏感**：英文搜索区分大小写（如"AutoCAD" ≠ "autocad"）。

---

## 缓存管理

每次建立索引会生成一个时间戳文件夹（如 `.read_price/cache/2026-06-08_15-18-42/`），包含该次索引的所有文件数据。

- **自动缓存**：索引结果自动保存，下次启动自动加载最新版本。
- **历史版本**：文件有改动时重新索引，旧版本保留不丢失。
- **清除缓存**：侧边栏提供"清除所有缓存"按钮。
- **重建索引**：侧边栏"重建索引"按钮，跳过缓存强制重新提取和 OCR。

## 环境要求与部署

### 环境要求
- **Windows 10/11**（64 位）
- **Python 3.12 64 位**（已加入 PATH）
- 首次使用需联网（下载依赖），之后可离线使用
- 可选：OpenAI 兼容大模型服务（用于 AI 整理）

### 首次安装
```
双击 setup.bat    （需联网，约 2-3 分钟）
```

### 日常使用
```
双击 start.bat    （自动检测兼容性，自动打开浏览器）
```

### 部署到新电脑
1. 复制项目文件夹（不含 `libs/`）到目标机器。
2. 确认已安装 **Python 3.12 64 位** 并加入 PATH。
3. 双击 `setup.bat`（需联网）。
4. 之后双击 `start.bat` 即可。

> `libs/` 中的二进制包与 Python 版本绑定，不能跨机器复制，必须在目标机器上运行 `setup.bat` 生成。

## 说明与限制

- **Word 文档没有物理页码**，.docx/.doc 只给出章节定位，页码显示为"—"。
- **api_key 明文存储**在 `.read_price/config.json`，请勿在公共电脑使用。
- **OCR 准确率**：扫描件 OCR 基于 RapidOCR（PP-OCRv4），对中文印刷体识别率高，手写体或严重倾斜的扫描件可能识别不准。
- **大文件处理**：500 页扫描 PDF 首次索引约需 5-10 分钟（取决于 CPU），之后从缓存加载秒级完成。

## 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `64-bit Python required, but 32-bit detected` | 安装了 32 位 Python | 安装 64 位 Python 3.12 |
| `libs/ is not compatible` | Python 小版本不匹配 | `start.bat` 会自动重建，或手动运行 `setup.bat` |
| `Python not found` | Python 未安装或未加入 PATH | 安装 Python 3.12 并勾选"Add to PATH" |
| `libs/ directory not found` | 未运行过 `setup.bat` | 运行 `setup.bat` |
| `localhost:8501 无法访问` | 防火墙拦截 | 运行 `netsh advfirewall firewall add rule name="Streamlit" dir=in action=allow protocol=tcp localport=8501` |
| 手动重建 `libs/` | 依赖损坏或需更新 | `python -m pip install --target libs --only-binary :all: -r requirements.txt` |
