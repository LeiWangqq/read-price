# 文件快速定位工具

在一批 PDF / Word 文档（含扫描件）中快速定位某个内容**在哪个文件、第几页、第几章**。基于 Streamlit 的本地网页应用，通过大模型（MiMo 等 OpenAI 兼容接口）进行智能定位。

## 功能
- 支持 **PDF**（含扫描件，自动 OCR）和 **Word(.docx)**。
- 两种文档来源：**指定本地文件夹**扫描，或**网页上传**文件。
- 输入自然语言查询（如"xx 在哪个文件里提到，第几页第几章"），返回命中的文件 / 页码 / 章节 / 原文片段。
- 首次使用在网页侧边栏填写大模型 `base_url` / `api_key` / `model`，本地保存，后续免填。

## 环境要求
- **Windows** + **Python 3.9 ~ 3.11**（PaddlePaddle 对 3.12+ 兼容性差，请勿使用）。
- 安装时需联网；PaddleOCR 首次运行会自动下载中文识别模型（约数十 MB），之后可离线使用。
- 一个可用的 **OpenAI 兼容**大模型服务（MiMo / DeepSeek / 通义 等），需要 `base_url`、`api_key`、`model` 三项。

## 安装与启动
1. 确认已安装 Python 3.9~3.11（命令行执行 `python --version` 验证）。
2. **双击 `start.bat`**。首次运行会自动 `pip install -r requirements.txt`（耗时较久），完成后自动启动并打开浏览器 `http://localhost:8501`。
3. 之后再次使用，直接双击 `start.bat` 即可（不会重复安装）。

> 若不想用 `start.bat`，也可手动执行：
> ```
> pip install -r requirements.txt
> streamlit run app.py
> ```

## 使用步骤
1. 首次进入，在左侧边栏填写大模型 `base_url`、`api_key`、`model`，点击保存。
2. 选择文档来源：
   - **本地文件夹**：填写文件夹路径，点击"扫描并建立索引"。
   - **上传文件**：拖入若干 PDF/Word，点击"建立索引"。
3. 在主区输入框输入查询，例如"防腐涂料的单价在哪个文件、第几页"，点击查询。
4. 结果以卡片展示：文件名、页码、章节、命中片段，可展开查看上下文。

## 说明与限制
- **Word 文档没有可靠的物理页码**，因此 .docx 只给出"章节"定位，页码显示为"—"。PDF 可精确到页。
- 配置保存在用户目录 `~/.read_price/config.json`（**api_key 为明文存储**，请勿在公共电脑使用）。
- 文档抽取/OCR 结果按"文件路径 + 修改时间"缓存到 `~/.read_price/cache/`，文件未变则不重复 OCR。

## 纯离线部署（可选）
在有网机器上先运行一次让 PaddleOCR 下载模型（默认在 `~/.paddleocr/`），再把该目录连同本程序拷贝到离线机器即可。
